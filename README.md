# 旅行计划应用

一个支持双人协作的旅行计划应用，具有卡片滑动浏览、留言和评分功能。

## 功能特点

- 📱 响应式设计，支持手机和桌面
- 🎴 卡片滑动浏览，直观的交互体验
- 💬 双人留言功能，可以互相交流
- ⭐ 评分功能，记录对每个行程项的评价
- ☁️ 云端同步，使用GitHub Gist实现数据同步
- 💾 数据导出/导入，支持本地备份

## 使用方法

### 基本使用

1. 打开 `index.html` 即可使用
2. 选择用户（用户A或用户B）
3. 浏览卡片，左右滑动查看不同行程项
4. 点击星星进行评分（1-5星）
5. 在留言框输入留言并发送

### 云端同步设置（推荐）

#### 第一步：创建GitHub Personal Access Token

1. 访问 [GitHub Settings > Developer settings > Personal access tokens > Tokens (classic)](https://github.com/settings/tokens)
2. 点击 "Generate new token (classic)"
3. 填写描述，例如："旅行计划同步"
4. 勾选 `gist` 权限
5. 点击 "Generate token"
6. **重要**：复制生成的token（只显示一次）

#### 第二步：配置同步

1. 在应用中点击 "⚙️ 配置" 按钮
2. 粘贴刚才复制的GitHub Token
3. Gist ID 留空（首次使用会自动创建）
4. 可选：勾选"启用自动同步"（每30秒自动上传）
5. 点击"保存配置"

#### 第三步：同步数据

- **上传**：点击 "☁️ 上传" 将本地数据上传到云端
- **下载**：点击 "⬇️ 下载" 从云端下载最新数据
- **自动同步**：如果启用了自动同步，数据会自动上传

### 导出/导入（备选方案）

如果不想使用GitHub同步，可以使用导出/导入功能：

1. **导出**：点击 "💾 导出" 下载JSON文件
2. **导入**：点击 "📥 导入" 选择之前导出的JSON文件

## 文件说明

- `index.html` - 主页面
- `data.js` - 行程数据配置（可修改此文件更换计划）
- `script.js` - 主要逻辑和交互功能
- `sync.js` - 数据同步功能
- `styles.css` - 样式文件

## 部署到GitHub Pages

1. 将所有文件推送到GitHub仓库
2. 在仓库设置中启用GitHub Pages
3. 选择主分支作为源
4. 访问 `https://你的用户名.github.io/仓库名/` 即可使用

## 注意事项

- GitHub Token 具有访问权限，请妥善保管
- Gist 数据默认是私有的，只有你可以访问
- 自动同步每30秒上传一次，可能会消耗一些API配额
- 建议定期使用导出功能备份数据

## 数据存储

- 本地数据存储在浏览器的 localStorage 中
- 云端数据存储在GitHub Gist中
- 每个卡片项的留言和评分独立存储

## 故障排除

### 同步失败

- 检查GitHub Token是否正确
- 确认Token有gist权限
- 检查网络连接
- 查看浏览器控制台的错误信息

### 数据丢失

- 检查是否清除了浏览器数据
- 尝试从云端下载数据
- 如果有导出备份，可以导入恢复
